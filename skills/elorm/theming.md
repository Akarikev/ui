# Theming

elorm/ui uses Tailwind CSS v4 with OKLCH CSS variables generated by `@elorm/themes`.

## Init presets

Set at `elorm init` (or in `elorm.json`):

| Setting | Options |
| --- | --- |
| Base color | neutral, zinc, slate, stone, gray |
| Accent | default, mono, blue, violet, green, orange, rose, amber, cyan |
| Radius | default (0.625rem), compact, round |

Marketing defaults: **neutral** base + **mono** accent.

## CSS structure

Global CSS (`tailwind.css` from config) includes:

```css
@import "tailwindcss";
@theme inline { /* extended tokens */ }
:root { /* light mode variables */ }
.dark { /* dark mode variables */ }
```

**Always edit the existing global CSS file** — never create a separate token file.

## Extended tokens

From `@elorm/themes`:

| Token | Purpose |
| --- | --- |
| `--surface-1/2/3` | Elevation scale (background → card → popover) |
| `--success/warning/info` | Status feedback |
| `--shadow-xs/sm/md/lg` | Elevation shadows |
| `--radius` | Base radius; scales to sm/md/lg/xl/2xl |

## Theme registry items

Maintainers build theme items automatically. Users can add:

```bash
npx elorm add theme-neutral
npx elorm add theme-accent-mono
```

These merge CSS variables into the global stylesheet.

## Runtime switching

Use `applyThemeToElement()` from `@elorm/themes` for live theme switching in demos or apps.

## Rules

- Use **semantic tokens** in components: `bg-background`, `text-muted-foreground`, `border-border`
- Never raw palette classes (`bg-blue-500`, `text-emerald-600`)
- No manual `dark:` color overrides — tokens handle light/dark
- `"style": "elorm"` in config (default) — future presets will extend this field
